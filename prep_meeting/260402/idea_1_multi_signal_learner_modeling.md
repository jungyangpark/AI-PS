# 아이디어 1: 다중 신호 기반 학습자 모델링

## 요약

현재 AI-PS 과제 제안서는 주로 **코드 편집 행동** (소요 시간, 편집, 커서 위치)에 의존하여 학습자의 지식 상태를 모델링함. 본 아이디어는 코드 편집 외의 **다중 보완적 신호**를 통합하여 학습자 모델을 확장함으로써, 학습자의 지식에 대한 더 풍부하고 정확한 실시간 이해를 구축하는 것을 제안함.

## 동기

단일 신호(코드 편집)는 노이즈가 많고 모호할 수 있음:
- 학생이 오랜 시간 멈춰 있다면 어려움을 겪고 있는 것인지, 깊이 생각 중인지, 아니면 단순히 딴짓을 하는 것인지 알 수 없음.
- 빠른 타이핑은 숙달을 의미할 수도 있고, 다른 곳에서 복사-붙여넣기하는 것일 수도 있음.
- 코드 정확성만으로는 학생이 개념을 진정으로 이해했는지 파악할 수 없음.

다중 신호를 결합하면 이러한 경우들을 구분하고, 자동완성을 제공하거나 제공하지 않을 때에 대한 더 나은 결정을 내릴 수 있음.

## 제안하는 추가 신호

### 신호 1: 자연어 질문 / 검색 쿼리
- **무엇을:** 학생이 코딩 중 검색하는 내용(Google, Stack Overflow, 문서)이나 AI 챗봇에 묻는 질문을 캡처.
- **이유:** 학생이 "재귀가 어떻게 작동하나요"를 검색한다면, 코드 편집에 아직 반영되지 않았더라도 해당 개념에 분명히 공백이 있음.
- **방법:** 브라우저 확장이나 IDE 통합 검색 패널로 쿼리 주제를 로깅; LLM을 사용하여 프로그래밍 개념별로 쿼리를 분류.
- **관련 논문:** Kim et al. (ACL 2025) — SQKT는 학생의 자연어 질문을 지식 추적에 통합하여 AUC 33.1% 향상 달성. (https://arxiv.org/abs/2502.10408)

### 신호 2: 오류 패턴 및 디버깅 행동
- **무엇을:** 컴파일 오류, 런타임 오류, 그리고 학생이 이에 어떻게 대응하는지(즉시 수정 vs. 동일 오류 반복 vs. 도움 요청) 추적.
- **이유:** 오류 패턴은 개념적 공백의 강력한 지표임. 동일한 유형의 오류(예: 반복문에서의 off-by-one)를 반복적으로 범하는 학생은 특정 지식 공백이 있음.
- **방법:** 컴파일러/린터 출력을 파싱하고, 오류를 개념별로 분류하고, 오류 해결 시간과 전략을 추적.
- **관련 논문:** Liang et al. (Scientific Reports 2022) — HELP-DKT는 해석 가능한 프로그래밍 지식 추적을 위해 오류 유형을 인코딩. (https://www.nature.com/articles/s41598-022-07956-0)

### 신호 3: 코드 구조 복잡도 변화
- **무엇을:** 학생 코드의 복잡도가 시간에 따라 어떻게 변화하는지 추적 (예: 중첩 깊이, 함수 수, 고급 패턴 사용).
- **이유:** 문제가 더 복잡한 구조를 요구하는데도 지속적으로 단순하고 평탄한 코드를 작성하는 학생은 해당 개념을 이해하지 못할 수 있음.
- **방법:** AST 분석 또는 srcML을 사용하여 시간에 따른 코드 스냅샷에서 구조적 특징을 추출.
- **관련 논문:** Shi et al. (EDM 2022) — Code-DKT는 지식 추적을 위해 code2vec으로 구조적 코드 특징을 추출. (https://educationaldatamining.org/edm2022/proceedings/2022.EDM-long-papers.5/index.html)

### 신호 4 (고급): 생리적 신호
- **무엇을:** 시선 추적 (코드 위의 응시 패턴), 표정, 또는 타이핑 리듬 패턴.
- **이유:** 인지 부하와 혼란은 생리적 상관관계가 있음. 특정 코드 줄을 장시간 응시하는 학생은 해당 개념에 어려움을 겪고 있을 수 있음.
- **방법:** 웹캠 기반 시선 추적 (예: WebGazer.js) 또는 타이핑 역학 분석.
- **참고:** 이 신호는 더 야심적이며 후속 연구에 더 적합할 수 있음.

## 통합 아키텍처

```
[코드 편집]    ──┐
[오류 로그]    ───┤
[검색 쿼리]    ──┼──→ [다중 신호 융합 모델] ──→ [개념별 지식 상태]
[코드 구조]    ──┤                                      │
[타이핑 리듬]  ──┘                                      ▼
                                              [자동완성 결정]
```

융합 모델의 형태:
- 가중 앙상블 (단순, 해석 가능)
- 신경망 모델 (예: 신호 임베딩의 어텐션 기반 융합)
- 다중 관측 유형을 갖는 베이지안 지식 추적 모델

## 단일 신호 대비 장점
1. **견고성:** 하나의 노이즈 있는 신호를 다른 신호가 보상할 수 있음
2. **조기 감지:** 검색 쿼리가 코드에 반영되기 전에 공백을 드러냄
3. **풍부한 개인화:** 학생마다 어려움의 다른 신호를 보임
4. **더 정확한 3단계 분류:** 초기 학습, 내재화, 숙달을 더 잘 구분

## 잠재적 도전 과제
- 검색/브라우징 행동 추적에 대한 프라이버시 우려
- 시스템 복잡성 증가와 잠재적 지연
- 다중 신호 모델 훈련을 위한 더 큰 데이터셋 필요
- 일부 신호(시선 추적)는 추가 하드웨어 필요

## 과제 제안서 타임라인과의 연관

- **1분기:** 데이터 수집 프로토콜에 검색 쿼리 및 오류 패턴 로깅 포함
- **2분기:** 학습자 모델링 알고리즘에 다중 신호 융합 구축
- **3분기:** VS Code 플러그인에 모든 신호 수집 구현
- **4분기:** 사용자 연구에서 다중 신호가 코드 편집만의 접근보다 우수한지 평가 (소거 연구)
