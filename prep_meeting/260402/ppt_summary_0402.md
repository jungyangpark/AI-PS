# AI-PS 프레임워크 기반 
## 미완성 코드 데이터 수집 및 시스템 구현 방안
**발표자:** 박정양
**날짜:** 2026. 04. 02.

---

# 1. 연구의 핵심 목표
- **목표:** '생산적 어려움(Productive Struggle)'을 유도하는 차등적 AI 코딩 어시스턴트 (AI-PS) 프레임워크 구축
- **Focus**
  1. 프로그래밍 개념 Knowledge Graph (KG) 구축
  2. 미완성 코드 및 에러 데이터 수집
  3. 학생의 지식 상태를 실시간 모델링 및 차등적 피드백 제공

---

# 2. 프로그래밍 개념 KG 구축
- **백본(Backbone) 모델:** CS-KG 2.0 활용
  - 논문 1,500만 건에서 개념간 관계(Relation Extraction)를 추출한 거대 지식 그래프.
- **한계점 및 해결 방안 (Hybrid Approach):**
  - 논문 기반 데이터이므로 '초보자의 기초 오개념' 노드가 부족할 가능성.
  - **해결 방안:** 상위 계층(배열, 반복문 등)은 CS-KG 2.0을 사용하되, 비어있는 하위 오개념 노드는 **표준 프로그래밍 전공서적 목차를 Ground Truth로 삼아 Human Labeling**으로 보완.

---

# 3. 데이터 수집 방향성 논의
실제 100명의 데이터를 수집하기 위한 시스템 환경 구축 방안. 
가장 현실적이고 효율적인 **두 가지 대안(Branch)**을 제안합니다.

1. **Branch 3-1:** VS Code 익스텐션 기반의 실제 행동 로그 수집
2. **Branch 3-2:** 기존 대규모 데이터셋 + LLM 기반의 인지적 오류 합성

---

# 4. Branch 3-1: VS Code 익스텐션 개발 (추천)
- **제안 배경:** 타겟 대상(대학생/전공자)의 실제 코딩 환경(IDE)과 동일하여 생태학적 타당성 높음.
- **엔지니어링 타당성:** 
  - 무거운 '웹 기반 샌드박스(코드 실행 백엔드)' 구축 비용을 제거하여 빠른 프로토타이핑 가능.
- **최종 산출물 연계:** 
  - 수집용 익스텐션이 향후 최종 AI-PS 서비스(MVP)의 뼈대로 그대로 재사용됨.

---

# 5. Branch 3-1: 기술적 구현 디테일
- **행동 로그 트래킹:** 
  - `onDidChangeTextDocument` API를 활용하여 수정/백스페이스 추적.
  - 디바운싱(Timer)을 적용해 '유의미한 고민 시간(Idle time)'만 묶어서 서버로 전송 (트래픽 최적화).
- **실행 에러(stderr) 맵핑:** 
  - 단순 타이핑뿐만 아니라 터미널의 에러 메시지를 캡처하여 코드 스냅샷과 연결.
- **경량화된 자기보고(Self-report) UI:** 
  - 장기 이탈 시 [로직 고민] / [구글 검색] / [휴식] 등을 클릭하는 팝업 UI 제공.

---

# 6. Branch 3-2: 합성(Synthetic) 데이터 활용 (대안)
- **제안 배경:** 대규모 인원 모집 및 시스템 구축 리스크를 줄이기 위한 Plan B.
- **구현 방식 (Truncation + Injection):**
  - MBPP, APPS 등 기존 대규모 데이터셋의 정답 코드의 뒷부분을 의도적으로 잘라냄(Truncation).
  - LLM 프롬프팅을 통해 '초보자가 할 법한 인지적 오류'를 주입하여 대규모 미완성 코드 합성.
- **타당성 검증:** 
  - 소규모(10~20명) 파일럿 실제 수집 데이터와 분포(Distribution) 비교를 통해 합성 데이터의 신뢰도 입증.

---

# 7. AI-PS 프레임워크 아키텍처
학생의 지식 상태를 3단계로 실시간 구분하기 위한 정량화 방안.

- **상태 구분:**
  - **숙달(Mastery):** 원활한 코드 작성
  - **내재화(Internalization):** 반복적 수정(Backspace) 및 잦은 에러 발생 (Struggle Index 상승)
  - **초기 학습(Initial):** 장시간의 타이핑 지연 (Idle Time)
- **학술적 근거 접목:** 
  - SQKT(ACL 2025) 및 Code-DKT(EDM 2022) 등 최신 지식 추적 알고리즘을 참고하여 모델링 고도화.

---

# 8. 차등적 개입 및 "안전장치(Safety Net)"
- **(1) 숙달 단계:** 생산성 극대화를 위한 전체 코드 자동완성 제공.
- **(2) 내재화 단계 (생산적 어려움):** 
  - 코드를 직접 짜주는 대신, 에러 위치에 밑줄만 긋거나 메타인지적 비계(질문형 힌트) 제공.
- **(3) 초기 학습 단계:** 명시적인 튜토리얼 및 큰 틀의 힌트 제공.
- **🚨 Frustration(좌절) 방지 안전장치:**
  - 과도한 인지적 부하(Cognitive Overload) 방지를 위해, 언제든 명시적 정답/힌트를 요청할 수 있는 **[On-demand Help] 버튼 UI** 필수 탑재.

---

# 9. 오늘 미팅 논의 사항 요약
1. **데이터 수집 시스템 방향 결정:** VS Code 기반(3-1) vs 데이터 합성(3-2)
2. **실험 타겟 명확화:** 전공자(CS 1학년 등) 위주인지, 완전 비전공자 포함인지 확인.
3. **KG 라벨링 기준:** 전공서적 기반 Human Labeling 방향성 동의 여부.
